NodeMCU 코드 요약

1. 주기적으로 각 센서의 습도와 Target 습도 정보를 서버에 요청하고 받음. 또한 위치정보도 서버에 요청하고 받음.

2. 사용자 어플리케이션으로 on/off 가능. -> 서버를 통해 on/off 여부를 확인

3. 총 4가지의 상태를 가지고 있음
	3-1. OFF 상태 : on이 입력될 때 까지 아무런 행동을 하지 않고, MOVE 상태를 제외한 상태에서 바로 OFF 상태로 들어갈 수 있음
		- off되면 가습기 현재위치 밑 "목적지 큐"(센서큐) 초기화
		- on되면 SLEEP 상태로 들어감
	3-2. SLEEP 상태 : 각 방의 습도정보와 사용자가 원하는 Target 습도를 서버로부터 받아와서 가습이 필요한 방을 큐에 넣는다.
		- 가습이 필요한 방을 "목적지 큐"에 집어넣고 큐에 들어온 순서대로 가습한다
		- 현재위치한 곳이 큐의 맨앞의 방이 같으면 바로 RUNNING 상태로 진입
		- 현재위치한 곳과 큐의 맨앞의 방이 다르면 목적지를 큐의 맨앞의 방으로 설정해 주고 MOVE 상태로 진입
	3-3 MOVE 상태 : 다른 방으로 이동할 때의 상태. 이동하는 중에는 목적지 방의 센서에 도착할 때 까지 이동상태에서 벗어나지 않는다.
		- 목적지 방에 도착하면 멈추고 RUNNING 상태로 진입한다.
	3-4 RUNNING 상태 : 가습을 실행한다. 현재 방의 습도가 사용자가 설정해준 Target 습도보다 높아질 때 까지 가습기가 동작한다.
		- 사용자가 설정해준 Target 습도보다 방의 습도가 낮아지게 되면 가습을 종료하고 SLEEP 상태로 들어간다.

+++++
4. 코드의 특정시간마다 실행되는 각 부분을 비동기적으로 실행하기 위해 delay() 함수를 사용하지 않고, 시간 변수를 만들어 사용하여 비동기적으로 처리함

5. 큐자료형을 사용하여 큐에 들어온 방 순서대로 가습이 가능. 추후 센서를 추가할 수 있게끔 구현하였음

6. MOVE 상태에서 다른 방으로 이동하기 위해 nodeMCU가 아닌 UNO보드와 통신을 하여 UNO보드에서 모터를 컨트롤 하게끔 nodeMCU에서 UNO보드를 제어함
	6-1. 다른 방으로 이동하는 것을 간략하게 보여주기위해 라인 트레이싱 기능을 이용하여 구현함
	6-2. nodeMCU에서 라인의 형태에 따라 UNO에 전진, 후진, 좌회전, 우회전등을 하도록 제어를 함
